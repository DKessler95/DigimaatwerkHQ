Integratie met Externe Services

Het Digimaatwerk CMS biedt mogelijkheden om te integreren met externe workflow automatiseringstools zoals Make.com (voorheen Integromat) en n8n.io. Deze integratie maakt het mogelijk om workflows te automatiseren wanneer content in het CMS wordt gepubliceerd, bewerkt of verwijderd.

Tip: Voor productieomgevingen wordt aanbevolen om een beveiligingstoken te configureren in de omgevingsvariabelen van de server om de webhook-integraties te beveiligen.

Integreren met Make.com

Make.com (voorheen Integromat) is een krachtige no-code automatiseringstool waarmee je verschillende applicaties en services aan elkaar kunt koppelen.

Stappen om te integreren met Make.com:

  1. Maak een nieuw scenario in Make.com
  2. Begin met een webhook trigger (HTTP-module)
  3. Configureer de webhook om JSON-data te ontvangen
  4. Configureer de endpoint URL: https://jouw-website.nl/api/webhooks/cms
  5. Voeg een Authorization header toe: 
    Bearer jouwGeheimeTokenHier
  6. Voeg verwerkingsstappen toe om de ontvangen CMS-data te verwerken
  7. Voeg acties toe, zoals:
    • Notificaties sturen (Email, Slack, etc.)
    • Data opslaan in databases
    • Content doorsturen naar sociale media
    • Updates naar andere systemen sturen

Integreren met n8n.io

n8n is een workflow automatiseringstool met een node-gebaseerde aanpak, waardoor je complexe workflows kunt bouwen met een visuele interface.

Stappen om te integreren met n8n.io:

  1. Start een nieuwe workflow in n8n
  2. Voeg een Webhook node toe als trigger
  3. Configureer de webhook voor POST-verzoeken en JSON-data
  4. Voeg een Function node toe om de ontvangen CMS-data te verwerken
  5. Voeg een HTTP Request node toe om te communiceren met API's van derden
  6. Voeg aanvullende nodes toe, afhankelijk van je automatiseringsbehoefte

Integreren met Zapier

Zapier is een populaire automatiseringstool die duizenden apps verbindt zonder dat je hoeft te coderen.

Stappen om te integreren met Zapier:

  1. Maak een nieuwe Zap aan in Zapier
  2. Selecteer "Webhooks by Zapier" als trigger
  3. Kies "Catch Hook" als triggergebeurtenis
  4. Configureer de webhook URL in het CMS of in je code
  5. Test de webhook door een voorbeeldpublicatie te maken
  6. Configureer de gewenste acties in Zapier

API Referentie

Het Digimaatwerk CMS biedt verschillende API endpoints die je kunt gebruiken voor integraties.

Webhook Payload

Wanneer content wordt gepubliceerd in het CMS, zal de volgende JSON-payload naar de geconfigureerde webhook URL worden verzonden: 

{
  "event": "publish",
  "collection": "blog",
  "slug": "voorbeeld-artikel",
  "locale": "nl",
  "title": "Voorbeeld artikel titel",
  "author": "Auteur Naam",
  "timestamp": "2025-03-30T12:00:00Z",
  "data": {
    // Complete content entry data
  }
}

Beschikbare API Endpoints

Endpoint Methode Beschrijving
/api/services GET Haalt alle services op
/api/blog GET Haalt alle blogposts op
/api/case-studies GET Haalt alle case studies op
/api/webhooks/cms POST Endpoint voor CMS webhook integraties
Let op: Zorg ervoor dat je API-endpoints beveiligd zijn met de juiste authenticatie voor productieomgevingen.
